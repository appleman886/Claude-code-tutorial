第一步：安装 Node.js

Node.js 是 Claude Code 的运行环境，必须先安装。

🪟 Windows 一键安装

步骤 1：打开命令提示符

  • 按下 Win + R 键
  • 输入 cmd 并回车
  • 或者按下 Win + X，选择"命令提示符"或"Windows PowerShell"

步骤 2：复制并运行安装命令

点击下方代码块复制命令，然后在命令提示符窗口中右键点击粘贴（会自动执行）：

winget install OpenJS.NodeJS.LTS

等待安装完成，期间可能需要你按 Y 确认。

步骤 3：验证安装

关闭当前命令提示符窗口，重新打开，输入以下命令：

node -v

如果显示版本号（例如 v20.x.x），说明安装成功！

💡 如果 winget 不可用

如果提示 winget 命令不存在，请访问 Node.js 官网 下载安装包，双击安装即可。

🍎 macOS 一键安装

步骤 1：打开终端

  • 按下 Command + 空格 打开 Spotlight 搜索
  • 输入"终端"或"Terminal"
  • 按回车打开终端应用
  • 或者在 Finder 中打开"应用程序" → "实用工具" → "终端"

步骤 2：检查是否安装了 Homebrew

在终端中输入以下命令，按回车：

brew --version

如果显示版本号，说明已安装。如果提示"command not found"，需要先安装 Homebrew。

步骤 3：安装 Homebrew（如果需要）

点击下方代码块复制命令，然后在终端窗口中粘贴（Command + V），按回车执行：

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

安装过程中需要你输入管理员密码（输入时不会显示字符），按回车确认。

步骤 4：使用 Homebrew 安装 Node.js

在终端中输入以下命令：

brew install node

等待安装完成。

步骤 5：验证安装

在终端中输入以下命令：

node -v

如果显示版本号，说明安装成功！

🐧 Linux 一键安装

步骤 1：打开终端

  • 按下 Ctrl + Alt + T 快捷键打开终端
  • 或者在应用菜单中搜索"Terminal"
  • 右键点击桌面或文件夹空白处，选择"在终端中打开"（不同发行版可能略有不同）

步骤 2：确认你的 Linux 发行版

在终端中输入以下命令查看系统版本：

cat /etc/os-release

查看输出中的 NAME 或 PRETTY_NAME 来确认你的发行版。

步骤 3：根据发行版选择安装命令

Ubuntu / Debian / Kali Linux:

curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt install -y nodejs

CentOS / RHEL / Fedora (RPM 系列):

curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs

注意：Fedora 可以使用 dnf 替代 yum

Arch Linux / Manjaro:

sudo pacman -S nodejs npm

Fedora:

sudo dnf install nodejs

openSUSE:

sudo zypper install nodejs

如何粘贴命令到终端：

  • 复制命令： 选中代码块中的命令，按 Ctrl + C 复制
  • 粘贴命令： 在终端中按 Ctrl + Shift + V 粘贴（大部分 Linux 终端）
  • 或者： 在终端中右键点击，选择"粘贴"
  • 或者： 有些终端支持 Shift + Insert 粘贴

步骤 4：安装过程中的提示

  • 如果提示输入密码，输入你的用户密码（输入时不会显示字符），按回车
  • 如果询问是否继续安装，输入 y 然后按回车
  • 等待安装完成，可能需要几分钟

步骤 5：验证安装

在终端中输入以下命令：

node -v

如果显示版本号（例如 v20.x.x），说明安装成功！

💡 常见问题

  • 提示权限不足： 在命令前加 sudo
  • curl 命令不存在： 先运行 sudo apt install curl 或 sudo yum install curl
  • 找不到对应发行版的命令： 访问 Node.js 官网 查看详细安装指南

第四步：配置智谱 AI API

智谱 AI 提供高性价比的模型服务，新用户有免费额度。

1. 获取智谱 AI API Key

步骤 1：访问智谱开放平台

访问 智谱开放平台，注册或登录账号。

访问智谱开放平台

步骤 2：创建 API Key

登录后，按照以下步骤获取 API Key：

  • 进入个人中心
  • 点击"项目管理"
  • 点击"API keys"
  • 创建新的 API Key
  • 复制生成的 API Key（妥善保管！）
选择您的操作系统：

当前配置针对：Windows

2. 配置环境变量

根据您的操作系统选择合适的配置方法。

方法一：使用 CMD（永久配置，推荐）

打开命令提示符，依次运行以下命令：

setx ANTHROPIC_AUTH_TOKEN "你的智谱 API 密钥"
setx ANTHROPIC_BASE_URL "https://open.bigmodel.cn/api/paas/v4"

将"你的智谱 API 密钥"替换为真实的 API Key

⚠️ 注意：需要重启命令行窗口才能生效。

方法二：使用 PowerShell（当前会话）

打开 PowerShell，运行：

$env:ANTHROPIC_AUTH_TOKEN="你的智谱 API 密钥"
$env:ANTHROPIC_BASE_URL="https://open.bigmodel.cn/api/paas/v4"

此方法仅在当前 PowerShell 会话有效，关闭窗口后失效。

方法一：使用 nano 编辑器配置（推荐）

步骤 1：确定配置文件

macOS 默认使用 Zsh shell，配置文件为 ~/.zshrc

echo $SHELL

如果输出 /bin/zsh 则编辑 ~/.zshrc，如果输出 /bin/bash 则编辑 ~/.bash_profile

步骤 2：打开配置文件

使用 nano 编辑器打开文件：

nano ~/.zshrc

步骤 3：添加环境变量

在文件末尾添加以下内容：

# 智谱 AI API 配置
export ANTHROPIC_AUTH_TOKEN="你的智谱 API 密钥"
export ANTHROPIC_BASE_URL="https://open.bigmodel.cn/api/paas/v4"

步骤 4：保存并退出

  • 按 Ctrl + O 保存文件
  • 按 Enter 确认文件名
  • 按 Ctrl + X 退出 nano

步骤 5：使配置生效

source ~/.zshrc

方法二：使用 echo 命令快速配置

如果不想手动编辑，可以直接运行以下命令：

echo 'export ANTHROPIC_AUTH_TOKEN="你的智谱 API 密钥"' >> ~/.zshrc
echo 'export ANTHROPIC_BASE_URL="https://open.bigmodel.cn/api/paas/v4"' >> ~/.zshrc
source ~/.zshrc

如果输出 /bin/bash 则编辑 ~/.bashrc，如果输出 /bin/zsh 则编辑 ~/.zshrc

步骤 2：打开配置文件

使用 nano 编辑器打开文件：

nano ~/.bashrc

步骤 3：添加环境变量

在文件末尾添加以下内容：

# 阿里云百炼 Coding Plan 配置
export ANTHROPIC_API_KEY="sk-sp-你的阿里云 API Key"
export ANTHROPIC_BASE_URL="https://coding.dashscope.aliyuncs.com/apps/anthropic"
# 可选：指定模型版本
# export ANTHROPIC_MODEL="claude-sonnet-4-5-20250929"

步骤 4：保存并退出

  • 按 Ctrl + O 保存文件
  • 按 Enter 确认文件名
  • 按 Ctrl + X 退出 nano

步骤 5：使配置生效

source ~/.bashrc

方法二：使用 echo 命令快速配置

echo 'export ANTHROPIC_API_KEY="sk-sp-你的阿里云 API Key"' >> ~/.bashrc
echo 'export ANTHROPIC_BASE_URL="https://coding.dashscope.aliyuncs.com/apps/anthropic"' >> ~/.bashrc
source ~/.bashrc

📝 Linux 特别提示

  • 权限问题：如果遇到权限错误，请确保当前用户有执行权限
  • 无头模式：Linux 服务器环境下完全支持，确保终端使用 UTF-8 编码
  • systemd 服务：如作为服务运行，需在 service 文件中配置环境变量

3. 验证配置

确保环境变量配置正确。

步骤 1：重启命令行窗口

关闭当前 CMD 或 PowerShell，重新打开新的窗口。

步骤 2：检查环境变量

echo %ANTHROPIC_API_KEY%
echo %ANTHROPIC_BASE_URL%

如果显示了对应的值，说明配置成功。

步骤 3：启动 Claude Code

claude
